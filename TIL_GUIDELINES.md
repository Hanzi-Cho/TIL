# TIL Repository Guidelines

이 문서는 `hanzicho/til` 저장소에서 Daily TIL 문서와 Knowledge 지식 문서를 작성하고 관리하는 AI의 작업 철학과 규칙을 정의합니다. AI는 이 저장소의 문서를 작성하거나 수정할 때 항상 이 가이드를 먼저 읽고 준수해야 합니다.

---

## 1. 핵심 철학 (Philosophy)
이 저장소는 단순히 개인 기록용이 아닙니다. **최종 목적은 AI가 축적된 지식 문서(Knowledge)를 바탕으로 사용자에게 모의 면접 질문(Grill)을 던져, 부족한 개념을 복습하고 학습을 확장해 나가는 선순환 구조를 만드는 것**입니다.

이 목적을 달성하기 위해 정보를 **개념 요약(Daily)**과 **상세 스펙(Knowledge)**으로 이원화하여 관리합니다.

---

## 2. 문서 분류 및 작성 규칙

### A. Daily 문서 (`daily/YYYY-MM-DD.md`)
- **역할**: 해당 날짜에 어떤 문제 상황(Situation & Task)을 만났고, 어떻게 행동(Action)하여 지식을 확장했는지의 "스토리"와 "목차"를 기록합니다.
- **작성 규칙**:
  1. **STAR 기법**을 적용하여 간략하게 작성하되, 구체적인 기술 스펙이나 긴 코드 스니펫은 적지 않습니다.
  2. 오늘 학습하고 정리한 **Knowledge 문서 수정 기록**을 표 형식으로 반드시 작성합니다.
  3. Knowledge 파일 경로는 반드시 마크다운 상대 경로 링크로 작성합니다: `[파일명](../knowledge/도메인/파일명.md)`

### B. Knowledge 문서 (`knowledge/도메인/개념.md`)
- **역할**: 기술의 내부 동작 원리(First Principles), 장단점, 핵심 개념, 코드 예제, 시각화 자료(Mermaid 등)를 집대성한 상세 지식 저장소입니다.
- **작성 규칙**:
  1. **상세성**: AI가 추후 이 문서를 기반으로 꼬리 질문(Deep Dive)을 던질 수 있도록 동작 원리와 예외 케이스를 자세히 기술합니다.
  2. **구조화**: 헤더 레벨(`##`, `###`)을 명확히 구분하고 표, 다이어그램, 코드 스니펫을 적극 활용합니다.
  3. **유지보수**: 새로운 인사이트나 수정사항이 생기면 기존 Knowledge 문서를 갱신(Update)하여 지식을 누적합니다.

---

## 3. AI 작업 흐름 (Workflow)
사용자가 새로운 학습 메모나 TIL 입력을 주면 AI는 다음과 같이 동작합니다:

1. **도메인 분류 및 파일 탐색**:
   - 입력된 개념들이 어떤 기술 도메인(예: `frontend`, `concurrency`, `react-native` 등)에 속하는지 파악합니다.
   - 기존 `knowledge/` 하위 디렉토리에 해당 개념을 다루는 문서가 있는지 확인합니다.
2. **Knowledge 문서 생성/수정**:
   - 기존 문서가 있다면 내용을 보완하여 갱신하고, 없다면 적절한 도메인 경로에 신규 문서를 생성합니다.
   - 기술의 **동작 원리, 문제 해결 코드 예시, Stale Closure/Memory Leak 등의 예외 케이스**를 상세히 기록합니다.
3. **Daily 문서 업데이트**:
   - 오늘 날짜의 `daily/YYYY-MM-DD.md` 파일에 오늘의 핵심 상황과 해결 스토리를 요약합니다.
   - 수정된 Knowledge 파일들을 하단의 `지식 문서 수정 기록` 표에 링크와 함께 추가합니다.
4. **인터뷰 연계**:
   - 작업 완료 후, 정리된 개념을 토대로 사용자에게 "다음 번 복습 시 어떤 수준의 질문(예: Event Loop의 내부 마이크로태스크 실행 우선순위 문제 등)을 던지면 좋을지" 제안하거나 가볍게 확인을 받습니다.
