# TIL [2026-07-05] - STAR 기법 적용

## Situation & Task (상황 및 문제 정의)
* **상황**: 매일 작성하는 TIL(Today I Learned) 문서를 Git 저장소에 커밋하고 푸시하는 과정이 번거롭고 누락될 위험이 있음.
* **목표**: 매일 특정 시간에 스스로 깃 커밋 및 푸시가 가동되도록 크론(Cron) 자동화를 시도하고, 최종적으로 가장 확실하고 직관적인 수동 실행형 래퍼 스크립트를 구축하는 것.

## Action (원인 분석 및 해결 과정)
* **1단계: 크론탭(Crontab) 스케줄러 등록**
  * 매일 밤 11시 50분(`50 23 * * *`)에 동작하는 자동 커밋/푸시 쉘 스크립트(`auto_push.sh`)를 작성하여 크론에 등록함.
* **2단계: 컴퓨터 전원 상태에 따른 크론의 한계 인식 및 피드백**
  * 개인 PC나 노트북이 꺼져 있거나 절전 모드(Sleep) 상태일 때 크론 스케줄이 무시된다는 사실을 검토.
  * `@reboot` 방식이나 `anacron` 대안이 있으나, 사용자의 요구에 맞게 크론 설정을 해제하고 `til` 프로젝트 루트 경로에서 `./push` 명령어 하나로 실행되는 래퍼 스크립트 구조로 전환하기로 결정.
* **3단계: 원격 커밋 충돌 방지 및 안전성 개선**
  * GitHub 원격 저장소에 주기적인 자동화 커밋(예: `skip ci`)이 비동기적으로 발생하여 로컬 push가 거절당하는 현상 발견.
  * `auto_push.sh` 내부에서 push를 진행하기 직전에 자동으로 원격 변경사항을 내려받아 합치도록 `git pull --no-rebase --no-edit origin main` 로직을 추가하여 안전한 선제 병합 처리.
  * `auto_push.log` 파일이 Git 추적 대상에 들어가지 않도록 `.gitignore` 파일을 작성하여 예외 처리.

## Result (결과 및 정량적 지표)
* 크론탭 스케줄러 의존성을 걷어내고, 사용자가 원하는 시점에 언제든 `./push` 명령어 단 한 번만으로 로컬 변경 사항 커밋, 원격 비동기 이력 병합, 원격 저장소 푸시까지의 3단계를 안전하게 자동 수행 완료.
* 불필요하게 생성되던 `auto_push.log` 파일도 Git 추적에서 깔끔히 격리됨.

## Takeaway (인사이트)
* 리눅스 `cron` 데몬은 예약 시간 당시 시스템이 활성화되어 있지 않으면 소급 적용하지 않고 스킵하므로, 항상 켜져 있는 서버가 아닌 개인 데스크톱/노트북 환경에서는 `@reboot`나 온디맨드 스크립트 실행이 더 직관적인 해결책이 될 수 있음.
* 협업 및 봇 자동화가 연동된 저장소 환경에서는 로컬 커밋을 강제로 올리기 전에 항상 원격 브랜치를 선제적으로 `pull` 하여 병합하는 방어적 스크립트 설계가 필수적임.
