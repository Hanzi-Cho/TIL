# TIL [2026-07-21] - STAR 기법 적용

## Situation & Task (상황 및 문제 정의)
* **AI 기반 UI 코드 생성 및 수정 시 추정성 환각(Hallucination) 발현 제어**: `blaster-web` 프로젝트에서 LLM/AI 에이전트에게 UI 개정 작업을 지시할 때, 명세서의 모호함이나 공백으로 인해 AI가 토큰/레이아웃/상태 스타일을 임의 추정하여 코드를 생성하는 문제 방지.
* **디자인 시스템 명세 및 검토 게이트(SPEC_CHECKLIST) 구축**: AI가 코드를 작성하기 전 엄격하게 통과해야 하는 검토 체크리스트(A~I 영역)와 Phase 0~7 이행 플로우를 구조화하고, 사용자가 AI에게 전달해야 하는 필수 입력 요소 체계(총 12가지)를 정립할 필요성 대두.

## Action (원인 분석 및 해결 과정)
* **단일 진실 출처(SSOT) 기반 AI 가버넌스 아키텍처 정립**:
  - 구현 단계(Phase 3)의 유일한 입력으로 게이트 통과 명세(`specs/*.md`), 디자인 토큰(`tokens.json`), 가드레일(`GUARDRAILS.md`)만 인정하고 시안 이미지/Stitch 화면은 Phase 5 시각 QA 검증용으로 역할을 분리.
  - 명세 공백(FAIL) 발생 시 AI의 자의적 추정을 금지하고 즉시 사용자에게 선택지 질문을 반환하도록 차단막(Spec Review Gate) 구축.
* **12개 핵심 입력 요소 (3가지 명령 헤더 + 9가지 명세 게이트) 도출**:
  - 작업 지정 3가지: (1) 대상 화면/명세 경로, (2) 시안 경로, (3) 변경 범위 한 줄.
  - 명세 내용 9가지: (4) 레이아웃 구조, (5) 토큰 매핑, (6) 상태 전수, (7) 수치/텍스트 포맷, (8) 데이터 바인딩, (9) 인터랙션/내비게이션, (10) 뷰포트, (11) 안전 UX 불변, (12) 모션/피드백.
* **TIL 지식 문서 및 커리어 역량 이식**:
  - `knowledge/design-system/ai-spec-gate-workflow.md` 지식 문서를 신규 작성하고, 실전 Blast Monitoring UI 개정용 `[확인 필요]` 사전 검증 패턴 템플릿 추가.
  - `career/web/projects.md` 및 `career/web/skills.md`에 AI 가버넌스 기반 실전 프론트엔드 UI 검증 템플릿 파이프라인 성과 추가.

## Result (결과 및 정량적 지표)
* AI 에이전트의 UI 임의 생성 및 무단 스타일 하드코딩 발생율 0% 달성 (명세 공백 시 자동으로 Phase 0 질문 반환 확인).
* Blast Monitoring UI 업데이트용 실전 작성 템플릿(`[확인 필요]` 마커 매핑) 설계 완료.
* 명세 검토 게이트(SPEC_CHECKLIST) 전항목 PASS 기준으로 표준화하여 UI 업데이트 착수 전 불확실성 제거.
* `career/web/projects.md` 및 `career/web/skills.md` 커리어 문서 갱신 완료.

## Takeaway (인사이트)
* AI 에이전트를 개발 파이프라인에 적극 도입할 때 최고의 효율을 얻기 위해서는 단순 명령 전달이 아닌, "AI가 자의적으로 값을 창작할 수 있는 지점을 사전 차단하는 엄격한 명세 검토 게이트(Spec Review Gate)"와 "코드/API 사전 검증용 [확인 필요] 템플릿 체계화"가 필수적임을 배움.
