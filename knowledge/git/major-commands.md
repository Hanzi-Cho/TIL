# Git 주요 명령어 지식 가이드 (Git Major Commands Guide)

이 문서는 실무 및 개인 프로젝트 소스 제어 시 필수적으로 마주하게 되는 Git 주요 명령어들의 한 줄 핵심 요약과 개인적 이해도를 바탕으로 분류한 학습 바인더입니다.

---

## 🟢 1. 설명 가능한 익숙한 명령어 (총 14종)

| 명령어 | 핵심 설명 및 동작 방식 |
| :--- | :--- |
| **`add`** | 작업 디렉토리의 변경 파일들을 다음 커밋 후보군인 스테이징 영역(Staging Area)에 기록합니다. |
| **`status`** | 현재 작업 디렉토리와 스테이징 영역에 있는 파일들의 상태(수정됨, 스테이징됨, 추적되지 않음 등)를 점검합니다. |
| **`tag`** | 릴리즈 버전(예: `v1.0.0`) 등 히스토리 상의 특정 중요한 커밋 시점에 영구적인 고유 이정표를 표시합니다. |
| **`init`** | 새로운 로컬 Git 저장소를 초기화하고 프로젝트 폴더 내에 `.git` 관리 디렉토리를 생성합니다. |
| **`commit`** | 스테이징 영역에 대기 중인 변경 파일들을 하나의 스냅샷으로 묶어 로컬 저장소 히스토리에 영구 기록합니다. |
| **`push`** | 로컬 저장소의 완료된 커밋 이력들을 지정한 원격(Remote) 저장소에 업로드하여 동기화합니다. |
| **`pull`** | 원격 저장소의 최신 커밋 이력을 다운로드(`fetch`)함과 동시에, 현재 작업 중인 로컬 브랜치에 자동으로 병합(`merge`)합니다. |
| **`checkout`** | 특정 커밋 시점, 다른 브랜치, 혹은 특정 버전의 파일 상태로 현재 작업 환경(Working Directory)을 전환합니다. |
| **`branch`** | 저장소 내의 브랜치를 신규 생성하거나 목록을 확인하고, 사용이 끝난 브랜치를 삭제하는 등 브랜치 흐름을 제어합니다. |
| **`stash`** | 현재 작업 중인 커밋되지 않은 임시 변경 사항들을 안전한 임시 보관소에 잠시 숨겨두고 워킹 트리를 깨끗하게 비웁니다. |
| **`config`** | Git 사용자의 이름, 이메일, 기본 에디터, 명령어 단축키(Alias) 등 전역/지역 환경 설정 정보를 다룹니다. |
| **`diff`** | 작업 디렉토리, 스테이징 영역, 혹은 두 커밋 간의 구체적인 소스 코드 변경 차이점을 비교 분석하여 출력합니다. |
| **`remote`** | 로컬 저장소와 연동할 원격 저장소(GitHub, GitLab 등)의 주소 및 연결(Alias) 정보를 등록하고 관리합니다. |
| **`merge`** | 다른 브랜치에서 분기되어 개발된 커밋 히스토리를 현재 브랜치로 가져와 두 브랜치의 흐름을 하나로 통합합니다. |

## 🟡 2. 개념 보완이 필요한 명령어 (총 13종)

| 명령어 | 핵심 설명 및 동작 방식 |
| :--- | :--- |
| **`clean`** | 작업 디렉토리 내에 존재하는 추적되지 않는(Untracked) 임시 파일들이나 자동 빌드 결과물들을 일괄적으로 삭제합니다. |
| **`blame`** | 특정 소스 파일의 각 라인을 누가, 언제, 어떤 커밋 메시지로 작성하고 수정했는지 라인 단위로 추적하여 분석합니다. |
| **`bisect`** | 이진 탐색(Binary Search) 알고리즘을 사용하여 버그가 최초로 유입된 문제의 커밋을 수많은 히스토리 중에서 신속하게 찾아냅니다. |
| **`submodule`** | 독립적인 외부 Git 저장소를 현재 진행 중인 프로젝트 저장소의 특정 하위 디렉토리에 링크하여 함께 버전을 추적합니다. |
| **`rebase`** | 현재 브랜치의 분기점(베이스)을 다른 브랜치의 최신 커밋 위치로 재설정하여, 머지 커밋 없는 깨끗한 단일 선형 히스토리를 만듭니다. |
| **`cherry-pick`** | 다른 브랜치에 들어있는 특정 커밋 단 하나만 지정하여 현재 작업 중인 브랜치에 똑같이 복제 적용합니다. |
| **`log`** | 현재까지 저장소에 누적된 커밋 히스토리(작성자, 날짜, 메시지 등)의 타임라인을 시간 역순으로 조회합니다. |
| **`reset`** | 특정 커밋으로 상태를 되돌리며, 옵션(`--hard/--mixed/--soft`)에 따라 작업 디렉토리와 스테이징 영역을 취소시킵니다. |
| **`restore`** | 커밋되지 않은 상태의 변경된 파일을 직전 커밋 상태나 스테이징 영역의 원본 상태로 안전하게 복원(되돌리기)합니다. |
| **`revert`** | 취소하고 싶은 커밋의 변경 사항을 **정반대로 적용하는 새로운 커밋**을 만들어, 기존 이력을 파괴하지 않고 안전하게 되돌립니다. |
| **`switch`** | 복잡한 `checkout` 명령어 기능 중 오직 '브랜치 전환(이동)' 기능만을 안전하고 직관적으로 사용하기 위해 분리된 명령어입니다. |
| **`reflog`** | 로컬 저장소에서 HEAD가 가리켰던 모든 참조 이동 기록을 보관하므로, `reset` 등으로 유실되거나 지워진 커밋도 복구할 수 있습니다. |
| **`fetch`** | 원격 저장소의 최신 커밋 데이터를 로컬로 다운로드만 해두고, 현재 작업 중인 로컬 코드 영역에는 바로 병합(`merge`)하지 않습니다. |
