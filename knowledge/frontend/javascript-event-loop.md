# 자바스크립트 이벤트 루프 (Event Loop)와 비동기 처리

이 문서는 자바스크립트(JS)의 싱글 스레드 환경에서 비동기 연산이 관리되는 내부 동작 원리인 **이벤트 루프(Event Loop)**와 태스크 큐의 우선순위를 정리합니다.

---

## 1. 이벤트 루프 기본 구조와 메모리 레이아웃

자바스크립트 엔진(예: V8)은 싱글 스레드로 동작하며, 하나의 콜 스택(Call Stack)과 하나의 힙(Heap)을 가집니다. 하지만 브라우저나 Node.js 런타임 환경은 멀티 스레드로 동작하며, 이벤트 루프와 여러 큐(Queue)를 통해 비동기 작업을 처리합니다.

```
+-------------------------------------------------------------+
| [브라우저 / Node.js 런타임]                                   |
|                                                             |
|  +------------------+         +--------------------------+  |
|  | 자바스크립트 엔진 |         | Web APIs (런타임 제공)   |  |
|  |                  |         |                          |  |
|  |  +------------+  |         |  DOM, setTimeout, fetch  |  |
|  |  | Call Stack |  |         |  database, I/O           |  |
|  |  +------------+  |         +-------------+------------+  |
|  +--------+---------+                       |               |
|           | (비동기 콜백 큐잉)              |               |
|           v                                 v               |
|  +-------------------------------------------------------+  |
|  | [큐 메모리 영역]                                      |  |
|  |                                                       |  |
|  |  * 마이크로태스크 큐 (Microtask Queue)                 |  |
|  |    [ Promise.then, queueMicrotask, MutationObserver ] |  |
|  |                                                       |  |
|  |  * 매크로태스크 큐 (Macrotask Queue / Task Queue)       |  |
|  |    [ setTimeout, setInterval, I/O, UI Rendering ]     |  |
|  +-------------------------------------------------------+  |
|                                                             |
|                         이벤트 루프                         |
|                 (콜 스택이 비면 큐에서 이동)                |
+-------------------------------------------------------------+
```

---

## 2. 마이크로태스크 vs 매크로태스크 (우선순위 메커니즘)

이벤트 루프가 비동기 태스크를 콜 스택으로 가져와 실행하는 순서는 다음과 같은 엄격한 우선순위 규칙을 따릅니다:

1. **콜 스택(Call Stack) 비우기**: 현재 실행 중인 동기 코드가 모두 실행되어 콜 스택이 완전히 빌 때까지 기다립니다.
2. **마이크로태스크 큐(Microtask Queue) 전체 처리**: 
   - 콜 스택이 비면, 이벤트 루프는 마이크로태스크 큐에 쌓여 있는 모든 콜백을 콜 스택으로 가져와 실행합니다.
   - **중요**: 마이크로태스크를 실행하는 도중에 새로운 마이크로태스크가 추가되면, 이벤트 루프는 대기열에 새로 추가된 마이크로태스크까지 **큐가 완전히 텅 빌 때까지 끝까지 다 처리**한 후에 다음 단계로 넘어갑니다.
3. **매크로태스크 큐(Macrotask Queue) 하나 처리**:
   - 마이크로태스크 큐가 완전히 비었음을 확인하면, 매크로태스크 큐에서 **가장 오래된 태스크 딱 하나(1개)**만 꺼내어 콜 스택으로 보낸 뒤 실행합니다.
   - 실행이 완료되면 다시 1단계로 돌아가서 콜 스택을 비우고 마이크로태스크 큐를 처음부터 다시 체크합니다.

### 💡 주요 작업 분류

- **마이크로태스크 (Microtasks)**:
  - `Promise.then()` / `Promise.catch()` / `Promise.finally()`
  - `async/await` (await 이후에 이어지는 코드)
  - `queueMicrotask()`
  - `MutationObserver` (브라우저)
  - `process.nextTick()` (Node.js 환경에서는 마이크로태스크보다도 우선순위가 높음)
- **매크로태스크 (Macrotasks / Tasks)**:
  - `setTimeout()` / `setInterval()` / `setImmediate()`
  - Network I/O, 파일 시스템 읽기/쓰기 등의 비동기 작업 콜백
  - UI 렌더링 및 인터랙션 이벤트 콜백

## 3. async / await의 내부 동작 원리

`async/await`는 새로운 비동기 처리 엔진이 아니라 `Promise` 체이닝을 가독성 있게 표현한 **문법 설탕(Syntactic Sugar)**입니다.

- **`await` 이전 코드**: 일반적인 동기(Synchronous) 코드와 동일하게 즉시 실행(Call Stack에 로드)됩니다.
- **`await` 이후 코드**: `await` 뒷부분에 명시된 비동기 작업이 완료된 후 실행될 나머지 코드 조각들은 `Promise.then()` 콜백 내부로 들어가 **마이크로태스크 큐로 등록**됩니다.

---

## 4. 코드 분석 퀴즈 (실행 순서 예측)

다음 코드의 콘솔 출력 결과를 예측하여 동작 구조를 파악합니다:

```javascript
console.log('1');

setTimeout(() => {
  console.log('2');
}, 0);

Promise.resolve()
  .then(() => {
    console.log('3');
  });

console.log('4');
```

### 실행 흐름 상세 분석:
1. `console.log('1')`이 동기적으로 실행되어 **`1`**이 출력됩니다.
2. `setTimeout`이 호출되면서 매크로태스크 큐에 콜백 `() => console.log('2')`가 추가됩니다.
3. `Promise.resolve().then()`이 호출되면서 마이크로태스크 큐에 콜백 `() => console.log('3')`이 추가됩니다.
4. `console.log('4')`가 동기적으로 실행되어 **`4`**가 출력됩니다.
5. 동기 코드가 모두 종료되어 콜 스택이 빕니다.
6. 이벤트 루프는 매크로태스크 큐로 가기 전, 마이크로태스크 큐에 쌓인 작업을 먼저 확인합니다. 큐에 들어있는 `() => console.log('3')`을 콜 스택으로 가져와 실행하여 **`3`**이 출력됩니다.
7. 마이크로태스크 큐가 완전히 비었으므로, 매크로태스크 큐에서 오래된 작업 하나(`() => console.log('2')`)를 가져와 실행하여 **`2`**가 출력됩니다.

### 최종 출력 결과:
```
1
4
3
2
```
*(마이크로태스크가 매크로태스크보다 항상 우선적으로 처리됨)*
