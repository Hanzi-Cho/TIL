# 현대자동차 SDV 직군 대비 차량용 소프트웨어 및 실시간 시스템 핵심 면접 질문 10선

이 문서는 SDV(Software Defined Vehicle) 및 전장 미들웨어 아키텍처 관점에서 차량 제어기 소프트웨어와 통신 인프라를 이해하기 위해 정리한 면접 질문 리스트입니다.

---

### Q1. 차량용 ECU(Electronic Control Unit) 통합과 아키텍처 변화
* 차량 내 ECU의 기능 정의와 기존의 분산형 제어기 구조의 한계를 공유형 아키텍처(Domain Controller, Zone Controller)와 함께 연동하여 설명하세요.
* **꼬리 질문**: 중앙 집중형 차량 컴퓨터(Central Compute Platform)로의 전환이 케이블 와이어링 하네스 무게 및 소프트웨어 업데이트 관점에서 주는 영향은 무엇인가요?

### Q2. CAN(Controller Area Network) 버스 통신 메커니즘
* CAN 통신 프레임 내에서 메시지 식별자(CAN ID)를 통한 버스 중재(Arbitration) 방식과 우선순위 판정 규칙에 대해 설명하세요.
* **꼬리 질문**: 두 개 이상의 노드가 동시에 송신할 때 데이터 충돌(Collision)이 하드웨어 레벨에서 회피되는 원리는 무엇이며, 데이터 처리량을 높이기 위해 설계된 CAN FD는 기존 CAN과 어떻게 다른가요?

### Q3. 차량용 이더넷(Automotive Ethernet)의 부각 배경
* 대용량 데이터 전송을 요구하는 현대 차량 아키텍처에서 기존의 대표적인 버스 통신(CAN, LIN, FlexRay, MOST) 규격들의 전송 대역폭 한계와 비교하여 차량용 이더넷이 표준 통신 프로토콜로 전환되는 구체적인 이유를 밝히세요.

### Q4. 무선 소프트웨어 업데이트(OTA, Over-The-Air) 파이프라인 설계
* 안전성이 보장된 차량 OTA 소프트웨어 수명 주기(다운로드 -> 무결성/암호화 검증 -> A/B 파티션 설치 -> 액티베이션 및 롤백 전략)의 백엔드와 단말 간 아키텍처를 설계해 보세요.
* **꼬리 질문**: 배포 및 설치 도중 차량 배터리 방전 등으로 시동이나 전원이 강제 오프되는 극한 시나리오에서의 장애 복구(Anti-Brick) 메커니즘은 어떻게 구성하나요?

### Q5. AUTOSAR(차량 소프트웨어 표준 플랫폼) 규격 분석
* 차량 소프트웨어 표준인 AUTOSAR의 도입 목적과 목적에 따른 두 가지 플랫폼 규격(Classic AUTOSAR vs Adaptive AUTOSAR)의 OS 커널 인터페이스, 통신 패러다임, 리소스 관리 기법 차이를 비교 대조하여 설명하세요.

### Q6. 실시간 운영체제(RTOS)의 핵심적 채택 이유
* 리눅스(Linux)나 안드로이드 등의 범용 OS와 대비되는 RTOS 커널(QNX, OSEK/VDX 기반 OS 등)의 결정론적 스케줄링(Deterministic Scheduling), 인터럽트 반응성(Latency) 및 컨텍스트 스위칭 비용 측면의 특성에 대해 설명하세요.
* **꼬리 질문**: '하드 실시간성(Hard Real-Time)'의 엄격한 준수가 차량 파워트레인이나 조향 제어 소프트웨어에서 생명 안전과 어떻게 직결되는지 정의하세요.

### Q7. 기능 안전 표준 ISO 26262와 안전 설계 방침
* 차량용 기능 안전 규격인 ISO 26262의 개요와 ASIL(Automotive Safety Integrity Level, A부터 D까지) 등급을 산정하는 3대 변수(Severity, Exposure, Controllability)를 설명하세요.
* **꼬리 질문**: 결함 감지 시 안전 모드로 전환하는 Fail-Safe 방식과, 결함이 있더라도 주행 제어를 임시 유지하는 Fail-Operational(자율주행용) 설계의 구체적 차이는 무엇인가요?

### Q8. 차량용 사이버 보안 요건과 하드웨어 레벨 보호
* 외부 통신 인터페이스가 다양해진 전장 환경을 보호하기 위해 설계하는 보안 파이프라인(Secure Boot, Firmware Signing, 전송 데이터 암호화)에 대해 설명하세요.
* **꼬리 질문**: CPU 하드웨어 단독 소프트웨어 검증의 취약성을 보강하기 위해 전장용 칩셋에 탑재되는 HSM(Hardware Security Module)의 역할은 무엇인가요?

### Q9. 차량용 멀티 센서 데이터 파이프라인 및 동기화
* 카메라(Camera), 레이더(Radar), 라이다(LiDAR) 등 이종 센서들로부터 입력되는 실시간 대용량 고속 데이터를 소프트웨어 계층(미들웨어)에서 지연 없이 취합하고 공간적/시간적 동기화(Sensor Fusion)하기 위해 사용하는 데이터 버퍼링 및 이벤트 큐 설계 전략을 설명하세요.

### Q10. 차량 내 시스템 성능 디버깅 및 지연 요소 디버깅 방법론
* 차량용 인포테인먼트(IVI/AAOS) 시스템의 부팅 후 첫 프레임 드랍이나 콕핏 디스플레이 반응성 저하와 같은 병목 문제가 보고되었습니다. 리액션 지연을 야기하는 잠재적 원인(Binder IPC 부하, SurfaceFlinger 병목, CAN/이더넷 버스 전송 지연, CPU 스위칭 정체 등)을 식별하기 위해 수립할 로깅 및 프로파일링 전략을 밝히세요.
