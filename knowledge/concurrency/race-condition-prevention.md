# Race

의도
blaster-launcher 프로젝트에서 
ai에게 명령을 내릴 때 설계 패턴을 제시하고, 
플로우 다이어그램과 시작, 종료 조건 오가는 데이터 포맷, 

꼬임 버그 원인들
- 시퀀스 오류: readiness, assertion, phantom type 등으로 타입으로 플로우 강제 
- 타이밍 오류: 
- 상태 불일치
- 외부 시스템 오류: 어댑터 잘 설계해서 오류났을 때 내 시스템이 손상되지 않고, 에러 핸들링을 원활히 수행하는 것이 목적.

# Data Race vs Race Condition
Data Race: 물리적 충돌
하나 이상의 쓰기가 포함된 2개 이상의 작업이 같은 메모리에 동기화 없이 접근하는 것.

Race Condition: 논리적 충돌
동기화를 제대로 처리하지 않고 여러 작업이 수행되어 논리적 오류를 일으키는 것.




이벤트 루프
콜 스택
